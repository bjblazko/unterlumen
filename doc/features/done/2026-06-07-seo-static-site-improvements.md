# SEO & Static Site Improvements

*Last modified: 2026-06-07*

## Summary

Transforms the published static gallery/site output into a search-indexable, semantically rich website by generating static HTML figure tags, adding structured metadata, and introducing human-readable album URLs.

## Details

### Slug-based album folders
Album directories are now named from the album title (`albums/paula-at-home/`) instead of an opaque postID (`albums/1c35123e52c1/`). Slugs are generated by `slugify()` (ASCII-safe, handles German umlauts) and deduplicated with a date suffix if needed. Stored in `site.json` as `slug`; albums without a slug fall back to `postID` for backward compatibility (no existing published content breaks).

### Static server-side figure tags
Previously `buildGallery()` in JavaScript injected `<figure>` elements at page load time. Now Go templates generate them statically in the HTML. Crawlers see all images without executing JS. The first two images per page use `loading="eager"` for better Largest Contentful Paint; the rest remain lazy.

### SEO metadata
- `<meta name="description">` — auto-generated ("A collection of N photos, January 2026").
- JSON-LD structured data: `ImageGallery` on album pages, `CollectionPage` on the root index.
- Better alt text: `"Album Title – Photo N of Total"` instead of the raw filename.
- Unique `<title>` on album pages: `"Album Title | Site Title"`.

### Canonical links + OG tags (optional)
When a **Site URL** is configured in channel settings (e.g. `https://example.com`), each page emits:
- `<link rel="canonical" href="...">`
- `og:title`, `og:description`, `og:image`

### robots.txt + sitemap.xml
- `robots.txt` generated in the site root on every publish/rebuild. Includes a `Sitemap:` line when Site URL is set.
- `sitemap.xml` generated when Site URL is configured; lists all album URLs as absolute URIs.
