# Crop Tool

## Summary

An interactive crop tool in the fullscreen viewer. The user draws a rectangle on the photo, chooses an optional aspect ratio, and saves the crop in-place. All metadata (EXIF, MakerNotes, Fujifilm film simulation, GPS) is preserved via exiftool.

## Details

- **Trigger**: "Crop" button in the viewer toolbar. Clicking it enters crop mode, replacing the toolbar with Cancel / aspect-ratio dropdown / Apply Crop.
- **Drawing**: Click-drag on the photo to draw a crop rectangle. Drag the box to reposition; resize with corner and edge handles. Rule-of-thirds grid displayed inside the box.
- **Aspect ratios**: Free, 1:1, 4:3, 3:4, 3:2, 2:3, 16:9, 9:16, 1.85:1 (Flat), 2.35:1 (Anamorphic), 2.39:1 (DCI Scope). Changing the ratio snaps the existing rectangle immediately.
- **Keyboard**: Escape cancels; Enter applies. Navigation keys (arrows, Delete) are blocked while crop mode is active.
- **Save**: Overwrites the original file in-place. The viewer reloads the image with a cache-bust query parameter after a successful crop.
- **Metadata**: `exiftool -TagsFromFile original -all:all` copies all metadata from the untouched original to the cropped temp file before the atomic rename. Dimension tags and Orientation are excluded (the crop bakes in the orientation). Fujifilm MakerNotes, film simulation tags, GPS, DateTimeOriginal, lens info — all survive.
- **Format handling**:
  - JPEG: Go `image/jpeg` decode + `applyOrientation` + `draw.Draw` crop + `jpeg.Encode(q=95)` + exiftool
  - PNG: Same with `png.Encode` (lossless)
  - GIF: Same with `gif.Encode`
  - WebP: `ffmpeg -vf crop -c:v libwebp -quality 90` + exiftool
  - HEIF/HEIC: `sips --cropToHeightWidth … --cropOffset` + exiftool. Requires macOS (`sips`). Returns an error on other platforms.
- **Coordinates**: Sent as fractions [0,1] of the visible image area. The overlay accounts for `object-fit: contain` letterboxing so fractions map precisely to image pixels. The backend applies EXIF orientation before cropping so the stored result matches what the user drew.
- **Aspect ratio math**: `this._ratio` is a visual pixel ratio (e.g. `1.0` = square pixels). Fractions map to pixels as `w_px = w_frac × imgW`, so the equivalent fraction-space ratio is `ratio × imgH / imgW`. The constraint uses this converted ratio, not the raw number, so 1:1 always renders as a visual square regardless of the image's own aspect ratio.

## Acceptance Criteria

- [ ] Crop button appears in viewer toolbar
- [ ] Entering crop mode replaces toolbar, hides nav buttons, shows overlay
- [ ] User can draw and resize crop rectangle with mouse
- [ ] Aspect ratio dropdown constrains the rectangle
- [ ] Escape cancels without saving
- [ ] Enter / Apply saves and reloads the image
- [ ] JPEG crop at quality 95 (no degradation beyond one generation loss)
- [ ] PNG crop is lossless
- [ ] HEIF crop preserved as HEIF (macOS, via sips)
- [ ] All EXIF metadata including Fujifilm film simulation survives
- [ ] Orientation-rotated images crop correctly (overlay maps to actual pixels)
- [ ] Error message shown in toolbar if crop fails
